## 容器重启后需要手动重新加载存档

**问题**

容器重启后，存档虽然可能已经被自动加载，但联机服务器组件没有完全初始化。此时玩家通常无法正常加入，直到你通过 VNC 手动重新加载一次存档。

**原因**

`Server Auto Load` 模组会在启动时通过自动流程加载存档，但在 Stardew Valley 1.6+ 中，联机网络层初始化不仅依赖存档本身，还依赖游戏内 Co-op 菜单的进入流程。自动加载绕过了这一初始化步骤，因此会出现“存档已加载但联机未完全就绪”的情况。

**临时解决方案**

容器重启后，按下面步骤恢复联机能力：

1. 通过 VNC 连接服务器，端口为 `5900`
2. 按 `ESC` 返回标题界面
3. 点击 `CO-OP`
4. 重新选择当前存档并点击加载
5. 等待游戏重新进入存档后，玩家即可正常加入

**耗时**

通常约 30 秒。

**影响**

- 不影响日常游玩，只会在容器重启后出现
- 不会造成数据丢失，存档本身仍然是安全的
- 健康检查通常仍会通过，因为游戏进程本身还在运行

**状态**

这是当前已知限制。要彻底解决，需要进一步修改 `Server Auto Load` 模组，使其在自动加载时也能正确初始化联机服务组件，这部分逻辑复杂，而且后续游戏更新时可能再次失效。

对大多数用户来说，当前 workaround 仍可接受，因为容器重启通常只发生在更新、维护或手动重启时。

## 日志中的音频警告

**问题**

你可能会在日志中看到类似下面的提示：

```text
OpenAL device could not be initialized
Steam achievements won't work because Steam isn't loaded
```

**原因**

服务器运行在无头环境中，没有真实音频设备，也不会像桌面 Steam 客户端那样完整加载图形与音频环境。

**影响**

这些通常只是警告，不会影响服务器核心功能。

**临时解决方案**

无需处理，可以安全忽略。

## 首次初始化必须使用 VNC

**问题**

第一次启动服务器时，你通常需要通过 VNC 手动创建存档，或者在游戏中手动加载一次现有存档。

**原因**

Stardew Valley 的联机服务依赖一个已经存在且被游戏内流程真正打开过的存档。首次部署时，游戏仍然需要通过图形界面完成一次初始进入流程。

**影响**

这是一次性的初始化要求。首次准备好存档后，后续大多数重启都可以依赖自动加载流程。

**临时解决方案**

按 README 中的引导完成首次设置：

1. 通过 VNC 连接到 `your-server-ip:5900`
2. 使用 `.env` 中配置的 `VNC_PASSWORD`
3. 进入 `CO-OP`
4. 创建新的联机农场，或者手动加载已有存档
5. 完成后如果不再需要图形界面，可以关闭 VNC 以节省资源
